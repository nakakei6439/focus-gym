# 老眼トレーニングアプリ 要件定義書（MVP版）

## 1. プロジェクト概要

### ■ アプリ名（仮）

FocusGym

### ■ 目的

老眼の進行抑制を目的とした視覚トレーニングの習慣化支援アプリ。

### ■ コンセプト

「1日3分、目のジム」

### ■ 対象OS

-   iOS
-   Android

------------------------------------------------------------------------

## 2. ターゲットユーザー

### ■ メインターゲット

-   38〜55歳
-   PC・スマホ作業が多い
-   近くの文字が見えにくくなってきた層
-   老眼鏡はまだ使いたくない層

### ■ ユーザー課題

-   何をすればよいかわからない
-   続かない
-   効果を実感できない

------------------------------------------------------------------------

## 3. 開発スコープ（MVP）

本バージョンでは以下のみ実装する：

-   トレーニング機能
-   継続管理機能
-   通知機能

※ 視力測定・AI分析・OCR機能は含まない

------------------------------------------------------------------------

## 4. 機能要件

### 4.1 トレーニング一覧機能

最低3種類を実装する。

#### ① 遠近ピント切替トレーニング

-   小さい文字を2秒表示
-   遠景風アニメーション表示へ切替
-   交互表示を3分間繰り返す
-   文字サイズ：3段階
-   切替スピード：3段階
-   実装方針：`AnimationController` + `Tween<double>` で文字サイズを交互にアニメーション

#### ② 追従運動トレーニング

-   画面上を動くオブジェクト（●）を目で追う
-   上下左右ランダム移動
-   速度：低・中・高
-   3分間実施
-   実装方針：`CustomPainter` + `AnimationController` で Canvas に ● を描画、`Tween<Offset>` でランダムな目標座標へアニメーション

#### ③ ぼかし→くっきり刺激トレーニング

-   ぼやけた文字を表示
-   徐々に鮮明化するアニメーション
-   ピント調整感覚を刺激
-   3分間実施
-   実装方針：`BackdropFilter` + `ImageFilter.blur` の sigma 値を `Tween<double>`（15.0 → 0.0）でアニメーション

------------------------------------------------------------------------

### 4.2 共通仕様

-   1セッション：3分固定
-   途中停止・再開可能
-   音声ガイド ON/OFF
-   終了後に達成画面表示

------------------------------------------------------------------------

## 5. 習慣化機能

### 5.1 継続管理

-   連続達成日数表示
-   月間カレンダー表示
-   累計トレーニング時間表示

### 5.2 リマインダー機能

-   通知時刻設定可能
-   1日1回通知
-   未実施時の再通知（1回）

### 5.3 達成演出

-   トレーニング完了アニメーション
-   励ましメッセージ表示
-   7日連続達成でバッジ付与

------------------------------------------------------------------------

## 6. 画面構成

ナビゲーション方式：BottomNavigationBar（ホーム / 履歴 / 設定）

| # | 画面名 | ルート | 備考 |
|---|--------|--------|------|
| 1 | スプラッシュ画面 | `/splash` | 起動時のみ表示 |
| 2 | ホーム画面 | `/home` | BottomNav タブ①、「今日やる」ボタンを大きく配置、連続日数表示 |
| 3 | トレーニング選択画面 | `/training` | 3種類のカード選択UI |
| 4 | トレーニング実施画面 | `/training/:type` | 60fps アニメーション、途中停止・再開可能 |
| 5 | 完了画面 | `/training/complete` | 達成演出・励ましメッセージ・バッジ表示 |
| 6 | 履歴画面 | `/history` | BottomNav タブ②、月間カレンダー・累計時間 |
| 7 | 設定画面 | `/settings` | BottomNav タブ③、通知時刻設定 |

------------------------------------------------------------------------

## 7. 非機能要件

| 項目 | 要件 |
|------|------|
| 起動時間 | 3秒以内 |
| 操作回数 | 3タップ以内で開始可能 |
| UI | 高コントラスト設計 |
| フォントサイズ | 大きめ標準設定（システムフォントスケール尊重） |
| フレームレート | 60fps維持 |
| データ保存 | 端末ローカル保存 |
| アーキテクチャ | Feature-first ディレクトリ構成 |
| 状態管理 | setState / InheritedWidget（Riverpod等は使用しない） |
| 最小OSバージョン | iOS 14以降 / Android 6.0（API 23）以降 |

------------------------------------------------------------------------

## 8. データ管理

### 保存項目

-   実施日
-   トレーニング種類
-   実施時間
-   連続達成日数

### 保存方式

-   ローカルデータベース（例：Hive / SQLite）

------------------------------------------------------------------------

## 9. KPI（評価指標）

-   D1継続率
-   7日継続率
-   月間平均実施回数
-   平均セッション完了率

## 10. 技術想定

| 用途 | 採用技術 | 備考 |
|------|----------|------|
| フロントエンド | Flutter 3.x（Dart 3.x） | iOS / Android 単一コードベース |
| ローカルDB | Hive（hive / hive_flutter） | TypeAdapter は hive_generator + build_runner で自動生成 |
| 通知機能 | flutter_local_notifications | Firebase不要・ソロMVP向け。将来的にFCMへ移行可 |
| 画面遷移 | go_router | 宣言的ルーティング |
| カレンダーUI | table_calendar | 履歴画面の月間カレンダー |
| コード生成 | build_runner | Hive TypeAdapter 自動生成 |

------------------------------------------------------------------------
